Phân tích - Ý tưởng

Khi mua M quyển, KH trả số tiền là `3000.M`, và được `M+1` quyển.

Đây là cách mua có lợi nhất. Vì vậy, KH nên mua từng nhóm (M+1) quyển như trên.

Số nhóm (M+1) quyển mua được nhiều nhất là k = N div (M+1) ⇒ số tiền = (3000*M)*k.

Số quyển lẻ (không đủ M+1) là le = N - k*(M+1) hoặc le = N mod (M+1)⇒ số tiền = 3000*le.

Thuật toán
+ Nhập N, M;
+ k = N div (M+1);
+ le = N mod (M+1);
+ tien = 3000*M*k + 3000*le;
+ Xuất tien;
Ghi chú - Bàn luận

Cách suy luận khác:

Cứ mua (M+1) quyển KH được giảm tiền của 1 quyển.

Vậy mua N quyển KH được giảm tiền của p = N div (M+1) quyển.

⇒ Số quyển KH phải trả tiền là `N-p` quyển.

+ Nhập N, M;
+ p = N div (M+1);
+ tien = 3000*(N-p);
+ Xuất tien;
